# 질문 답변 카드 매칭 서비스 - 요구사항

## 프로젝트 개요
카드 스크롤 페이지에서 사용자가 카드를 선택하면, 미리 준비된 질문과 답변자들을 랜덤으로 매치하여 보여주는 서비스

---

## 주요 데이터

### 1. 질문 (Questions)
- **총 28개**의 미리 준비된 질문
- 각 질문은 고유한 ID와 텍스트를 가짐

### 2. 답변자 (Respondents)
- **총 7명**의 답변자
- 각 답변자는 다음 정보를 포함:
  - 이름 (Name)
  - 사진/이미지 (Image/Profile Photo)

---

## 주요 기능

### 카드 선택 시 동작
1. 사용자가 카드 스크롤 페이지에서 카드 하나를 선택
2. 다음 정보가 랜덤으로 매치되어 표시됨:
   - **질문 1개** (랜덤 선택)
   - **답변자 2명** (랜덤 선택)

---

## 핵심 제약 조건

### ✅ 질문 중복 방지
- **한 번 뽑은 질문은 다시 뽑을 수 없음**
- 게임 진행 중 사용한 질문은 추가 선택 시 제외
- 예: 1번째 선택에서 질문#5를 뽑으면, 그 이후 모든 선택에서 질문#5는 제외

### ✅ 답변자 연속 선택 방지
- **직전(N-1)에 뽑힌 답변자는 다음(N)에 다시 뽑히지 않음**
- **2번 전(N-2)에 뽑힌 답변자는 다음(N)에 다시 뽑힐 수 있음**

**예시:**
```
1번째 선택: 답변자 A, B 선택 → 결과 표시
2번째 선택: A, B 제외하고 C, D, E, F, G 중 2명 선택
3번째 선택: C, D 제외하고 (2번째 선택자) 
           A, B는 다시 선택 가능 (1번째 선택자이므로)
           E, F, G 중 2명 선택
```

---

## 게임 상태 관리

### 추적 필요 정보
1. **사용된 질문 목록** (전체 게임 기간)
2. **이전 선택 답변자** (1턴 이전)
3. **현재 선택 답변자** (현재 턴)

### 게임 종료 조건
- 질문 28개가 모두 사용되었을 때
- 더 이상 새로운 질문을 선택할 수 없음

---

## UI/UX 플로우

1. **카드 스크롤 페이지**: 사용자가 여러 카드 중 하나를 선택
2. **결과 표시 페이지**: 선택된 카드에 매칭된 내용 표시
   - 선택된 질문
   - 매칭된 2명의 답변자 (이름 + 사진)
3. **다시 선택**: 버튼을 눌러 다음 카드 선택으로 진행

---

## 기술 구현 고려사항

- 질문 배열: 사용된 질문을 추적하고 필터링
- 답변자 선택: 이전 턴의 답변자를 메모리에 유지하고 제외
- 상태 관리: React State 또는 Context API를 통해 게임 상태 관리
